====================================================
  banking77_finetune.ipynb — Run Instructions
====================================================

OVERVIEW
--------
Fine-tunes Llama-3.2-1B-Instruct on the Banking77 77-class intent
classification dataset. Uses QLoRA + rsLoRA + NEFTune on a free Colab T4 GPU.
Full training: ~50 min. Evaluation (Section 11): ~25 min additional.

--------------------------------------------------
ONE-TIME SETUP (do this before your first run)
--------------------------------------------------

1. Create a W&B account (free): https://wandb.ai
   - Copy your API key from: https://wandb.ai/settings → API Keys

2. Add the API key to Colab Secrets:
   - Open the notebook in Colab
   - Click the key icon (🔑) in the left sidebar → "Secrets"
   - Click "+ Add new secret"
   - Name:  WANDB_API_KEY
   - Value: <your W&B API key>
   - Toggle "Notebook access" ON

   Without this, the W&B login cell will fall back to an interactive prompt
   (you'll be asked to paste the key manually on first run).

3. Get a Google account with Google Drive (needed for checkpoint storage).


--------------------------------------------------
FIRST-TIME RUN
--------------------------------------------------

1. Open banking77_finetune.ipynb in Google Colab
   (File → Upload notebook, or open from Drive)

2. Set runtime:
   Runtime → Change runtime type → Hardware accelerator: T4 GPU → Save

3. Verify the runtime: the first cell (Section 0) runs:
       !nvidia-smi
   You should see T4 GPU with ~15 GB free VRAM. If not, disconnect and
   reconnect to get a fresh runtime (Runtime → Disconnect and delete runtime).

4. Check the two run-mode flags in the Config cell (after the Drive mount cell):
       RESUME_MODE = False   ← keep False for a fresh training run
       RUN_SWEEP   = False   ← keep False unless you want the 75-min sweep

5. Click: Runtime → Run all

6. When prompted by Google Drive, click "Connect to Google Drive" and
   authorise in the popup. This is the ONLY manual interaction required
   during a full run.

7. Expected console output during training (Section 6):
       {'loss': 1.xyz, 'grad_norm': ..., 'learning_rate': ..., 'epoch': 1.0}
       [GenerationEval] Epoch 1: Accuracy = 0.xxxx (xxx/500)
       [TimeBasedCheckpoint] Saved checkpoint ...
       ...
   Training runs for 3 epochs (~50 min). W&B dashboard updates live.

8. After training completes, Section 11 runs automatically:
   - Full batched evaluation (all 3,076 test samples, ~20 min)
   - Per-class accuracy table
   - Top confused pairs analysis
   - Trie-based constrained decoding evaluation
   All results are logged to W&B as sortable tables.

Output locations in Google Drive:
   MyDrive/banking77_finetune/
   ├── models/final_lora/          ← trained adapter (best epoch by eval_loss)
   ├── checkpoints/                ← per-epoch HuggingFace checkpoints
   ├── checkpoints/periodic_backup/← 30-min safety snapshots
   └── logs/                       ← TensorBoard event files


--------------------------------------------------
RESUMING AFTER A DISCONNECT
--------------------------------------------------

Colab sessions disconnect after ~90 min idle or ~12 hr total. The
TimeBasedCheckpointCallback saves every 30 min to Drive, so you lose at most
30 min of training.

To resume:
1. Reconnect to Colab (Runtime → Reconnect)
2. Set RESUME_MODE = True in the Config cell
3. Run all cells from the top (Runtime → Run all)
   - The Drive mount and setup cells will re-run (fast, idempotent)
   - The resume cell (Section 9) will auto-load the latest Drive checkpoint
   - Training will continue from where it left off

Do NOT set RESUME_MODE = True if you want a fresh run — it will overwrite
nothing, but it wastes time finding a non-existent checkpoint.


--------------------------------------------------
RUNNING THE HYPERPARAMETER SWEEP (optional)
--------------------------------------------------

The sweep trains 5 configurations for 1 epoch each (~75 min total).
Bayesian optimisation over {learning_rate, lora_rank, batch_size}.

Prerequisites:
- Complete a full training run first (RESUME_MODE=False, RUN_SWEEP=False)
  so the base dataset and tokenizer are cached

To run the sweep:
1. Set RUN_SWEEP = True in the Config cell
2. Runtime → Run all
   - All training cells run normally first
   - Section 10 sweep cells execute after training completes
   - Results appear in W&B under project "banking77-sweep"

To run ONLY the sweep (skip re-training):
1. Set RESUME_MODE = True  (load trained model)
2. Set RUN_SWEEP   = True
3. Runtime → Run all

LOADING THE TRAINED ADAPTER (outside Colab)
--------------------------------------------------

from peft import PeftModel
from transformers import AutoModelForCausalLM, AutoTokenizer

base  = AutoModelForCausalLM.from_pretrained("unsloth/Llama-3.2-1B-Instruct")
model = PeftModel.from_pretrained(base, "/path/to/MyDrive/banking77_finetune/models/final_lora")
model = model.merge_and_unload()   # optional: merge weights for faster inference
tokenizer = AutoTokenizer.from_pretrained("/path/to/MyDrive/banking77_finetune/models/final_lora")

Replace /path/to/ with the actual path to your Google Drive folder on the
machine where you downloaded the adapter.


--------------------------------------------------
TROUBLESHOOTING
--------------------------------------------------

PROBLEM: nvidia-smi shows no GPU / "No GPU found"
SOLUTION: Runtime → Disconnect and delete runtime → Reconnect → Change
          runtime type to T4 GPU. Free Colab T4 availability varies; try
          again in a few minutes if none are available.

PROBLEM: "CUDA out of memory" during training
SOLUTION: The notebook auto-detects fp16/bf16. If OOM still occurs:
          - Reduce per_device_train_batch_size from 8 to 4
          - Reduce max_seq_length from 512 to 256
          Both settings are in the SFTConfig cell (Section 6).

PROBLEM: W&B login cell prompts for API key interactively
SOLUTION: You haven't set WANDB_API_KEY in Colab Secrets. See ONE-TIME SETUP
          above. Alternatively, paste your key when prompted — it only needs
          to be done once per Colab session.

PROBLEM: Google Drive auth popup doesn't appear
SOLUTION: Re-run the Drive mount cell (Section 2) manually. If the popup is
          blocked by your browser, disable the popup blocker for colab.research.google.com.

PROBLEM: Training runs but W&B shows no metrics / "wandb: Run data is saved locally"
SOLUTION: The wandb.init() call failed silently. Check that your API key is
          valid at https://wandb.ai/settings. Re-run the W&B login cell.

PROBLEM: "ModuleNotFoundError: No module named 'unsloth'"
SOLUTION: The install cell (Section 1) did not finish. Re-run it manually
          (it takes ~3 min). Then re-run all subsequent cells.

PROBLEM: Checkpoint save fails with "Permission denied" or Drive quota error
SOLUTION: Check that you have available Drive storage. The model adapter
          (adapter_model.safetensors) is ~50 MB; full checkpoints are ~500 MB.
          Delete old periodic_backup snapshots from Drive if needed.

PROBLEM: Section 11 (full eval) runs out of memory
SOLUTION: Reduce the batch_size in run_full_eval from 32 to 8 or 16. Edit
          the function call at the bottom of the Section 11 setup cell.

PROBLEM: Constrained decoding is much slower than expected
SOLUTION: The trie lookup adds ~1.5× overhead. For 3,076 samples this takes
          ~25 min. This is expected. Do not interrupt — it will finish.

PROBLEM: Resume cell loads wrong checkpoint (old run instead of latest)
SOLUTION: The resume cell finds the checkpoint with the newest timestamp in
          MyDrive/banking77_finetune/checkpoints/periodic_backup/. If you have
          checkpoints from multiple runs, delete the old ones from Drive.

EXPECTED TRAINING OUTCOMES
--------------------------------------------------

  GenerationEval accuracy (epoch 3)   ~88–92%
  Full eval exact-match accuracy      ~88–92%
  Constrained decoding accuracy       ~88–92% (+0.05–0.2pp over free gen)

Actual result from reference run:
  Full eval:          90.21% (2775/3076)
  Constrained:        90.28% (2777/3076)  — +0.07pp, 2 fixed, 0 broken

If your accuracy is significantly lower:
  - Verify the T4 GPU is connected (Section 0)
  - Check that packing=True is set in SFTConfig (critical for training speed)
  - Verify load_best_model_at_end=True loaded the best epoch checkpoint
